## 概要

STModuleManagerは **PCソフト版** と **Web版** の2種類を提供しています。

### PCソフト版

インストール不要で、ダウンロードして展開するだけで使用可能です。ゲーム内の通信をキャプチャしてモジュール一覧を自動取得し、モジュールの追加・削除といった変更も自動で反映されるため、煩雑な手入力なしでモジュール情報を管理できます。さらに、取得した情報をもとに最適な4モジュールの組み合わせをアルゴリズムで自動探索する機能も搭載しており、複数の条件を切り替えながらベストな組み合わせを短時間で確認できます。

アルゴリズムの詳細については [最適化の計算方法](#最適化の計算方法) を参照してください。

**[Releases](../../releases) ページから最新の `StarResonanceModuleTool-vX.X.X.zip` をダウンロードし、任意のフォルダに展開してください。**

**動作要件:** Windows 10 / 11（パケットキャプチャに管理者権限が必要）

> [!NOTE]
> 本ツールはパケットキャプチャに [WinDivert](https://reqrypt.org/windivert.html) を使用しているため、Windows Defender などのアンチウイルスソフトが誤検知する場合があります。その際はフォルダを除外設定に追加してください。

### Web版

スクリーンショットや画面キャプチャからOCRでモジュール情報を読み取り、データとして取り込めます。最適化ロジックはPCソフト版と共通化されているため、同じ最適化機能をブラウザ上でも利用できます。パケットキャプチャが不要なため、スマートフォンを含むあらゆる端末から手軽にご利用いただけます。

**[Web版を開く](https://st-module-manager.pages.dev/)**（スマホ対応）

## 機能

### 共通

- モジュール一覧の表示・多条件フィルタリング（AND/OR切替）
- 複数キーのカスタムソート（昇順/降順切替）
- 最適な4モジュールの組み合わせを自動探索する最適化機能
  - メイン / サブ / 除外ステータスの個別設定
  - 探索速度の選択（標準 / 高精度 / 最高精度 / 総当たり）
  - 最適化パターンの保存・読込
- JSON エクスポート（PC版・Web版で互換）

### PCソフト版固有

- ゲーム内通信のパケットキャプチャによるモジュールデータの自動取得
- バックグラウンドモード（最小化でトレイへ移行）
- テーマ切替（システム連動 / ライト / ダーク）
- CSV エクスポート

### Web版固有

- スクリーンショット / 画面キャプチャからOCRでモジュール情報を取り込み
  - ドラッグ&ドロップ・クリップボード貼り付けに対応
  - 連続キャプチャ & バックグラウンドOCR処理（Chromium系ブラウザ）
- スマホ・タブレット対応のレスポンシブUI

## 最適化の計算方法

### 概要

手持ちモジュール（最大約2000個）から4つを選び、指定ステータスの**ブレイクポイント到達**と<strong>+値合計</strong>を最大化する組み合わせを求めます。

### ブレイクポイント

1つのステータスの合計値（4モジュール合算、最大+20）に応じて効果が段階的に上昇します。

| 合計値 | 効果 |
|--------|------|
| +1 | 発動 |
| +4 | 上昇 |
| +8 | 上昇 |
| +12 | 上昇 |
| +16 | 大幅上昇 |
| +20 | 最大 |

### ステータスカテゴリの選び方

| カテゴリ | 意図 |
|---------|------|
| **メイン** | +20到達を最優先で狙うステータス。ビルドの軸として必ず設定します |
| **サブ** | メインを確保した上で、余裕があれば+16以上を目指したいステータス |
| **非選択** | 優先度は低いものの、ついていれば損はない程度のステータス。<br>完全には無視されず、わずかにスコアへ加算されます |
| **除外** | ビルドに不要で、あっても恩恵がないステータス。<br>スコアに一切加算されないため、除外ステータスばかりのモジュールは選ばれにくくなります |

### スコア計算

各ステータスごとに、到達したブレイクポイントの**最高段階**に応じて点数が加算されます。

| ブレイク<br>ポイント | メイン<br>(× 1.0) | サブ<br>(× 0.3) | 非選択<br>(× 0.1) | 除外 |
|-----------------|----------------|--------------|----------------|------|
| +20 到達 | 10,000 pt | 3,000 pt | 1,000 pt | 0 |
| +16 到達 | 5,000 pt | 1,500 pt | 500 pt | 0 |
| +12 到達 | 100 pt | 30 pt | 10 pt | 0 |
| +8 到達 | 50 pt | 15 pt | 5 pt | 0 |
| +4 到達 | 20 pt | 6 pt | 2 pt | 0 |
| +1 到達 | 5 pt | 1.5 pt | 0.5 pt | 0 |

これに加えて、**全ステータスの+値合計 × 2** がスコアに加算されます。

> **優先順位:** +20到達 > +16到達 > +値合計 > +12以下のブレイクポイント
>
> +20 / +16 の点数を+値ボーナスの理論最大値より十分大きく設定することで、ブレイクポイント到達が常に優先されます。

### 探索の高速化

C(2000, 4) ≈ 6,650億通りを全探索するのは現実的でないため、以下の手順でフィルタリングしています。

1. **関連性フィルタ** — メイン/サブステータスを1つも持たないモジュールを除外
2. **レアリティフィルタ** — 指定品質未満のモジュールを除外
3. **貢献度スコア Top N** — 各モジュールの貢献度（`Σ(メイン値×3) + Σ(サブ値×1) + Σ(その他値×0.5)`）上位N件に絞り込み

候補数Nは探索精度の設定で変更できます。

| 設定 | 候補数 | 説明 |
|------|--------|------|
| 標準（デフォルト） | 200 | 高速で十分な精度 |
| 高精度 | 300 | より広い探索範囲で精度向上 |
| 最高精度 | 600 | 最も精度が高いが時間がかかる |
| 総当たり | 全件 | フィルタなしの完全探索（後述） |

#### 総当たりモード

> [!CAUTION]
> 総当たりモードは**処理に最大数時間かかる場合があります**。通常の使用では標準〜最高精度で十分な結果が得られるため、基本的にはおすすめしません。
>
> 関連性フィルタ・貢献度スコアによる候補絞り込みを行わず、レアリティフィルタのみを適用した上で全モジュールを対象に探索します。理論上最も正確な結果が得られますが、モジュール数が多いほど組み合わせ数が爆発的に増加するため、非常に長い処理時間が必要です。
>
> どうしても最適解を見逃したくない場合や、候補数の絞り込みによる漏れがないか検証したい場合にのみご利用ください。

フィルタ後、Rayonによる**マルチコア並列探索**と**枝刈り**（2つ選んだ時点で残りの各ステータスに+20を加算した楽観的上界スコアが現在のベストを下回る場合に打ち切り）を組み合わせることで、CPU上で現実的な時間内での探索を可能にしています。
