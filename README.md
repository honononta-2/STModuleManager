# スタレゾのモジュール管理ソフト

[![GitHub Release](https://img.shields.io/github/v/release/honononta/STModuleManager)](https://github.com/honononta/STModuleManager/releases/latest)
[![GitHub Downloads](https://img.shields.io/github/downloads/honononta/STModuleManager/total)](https://github.com/honononta/STModuleManager/releases)

**[ブループロトコル：スターレゾナンス](https://bpsr.xdg.com/jp/)（スタレゾ）のモジュールを管理・最適化する非公式ツールです。**

> [!WARNING]
> 本ツールはゲーム運営とは一切関係のない非公式ツールです。
> 使用により発生したいかなる問題についても、開発者は責任を負いません。
> **すべて自己責任でご使用ください。**

## Web版

**[Web版はこちら](https://st-module-manager.pages.dev/)**（スマホ対応）

スクリーンショットからモジュールをOCR取込し、最適化を実行できます。パケットキャプチャ不要で、どの端末からでも利用可能です。

> [!TIP]
> スクショ取込は一度に大量に行うとブラウザが落ちる場合があります。数枚ずつに分けて取り込んでください。

## 機能

- ゲーム内モジュールデータの自動取得（パケットキャプチャ）
- モジュール一覧の表示・フィルタリング・ソート
- 最適な4モジュールの組み合わせを自動探索する最適化機能
- JSON / CSV エクスポート

## ダウンロード

[Releases](../../releases) ページから最新の `StarResonanceModuleTool-vX.X.X.zip` をダウンロードし、任意のフォルダに展開してください。インストール作業は不要です。

### 動作要件

- Windows 10 / 11
- 管理者権限（パケットキャプチャに必要）

> [!NOTE]
> 本ツールはパケットキャプチャに [WinDivert](https://reqrypt.org/windivert.html) を使用しているため、Windows Defender などのアンチウイルスソフトが誤検知する場合があります。その際はフォルダを除外設定に追加してください。

## 使い方

### モジュールデータの取得

1. **スタレゾを開く** — ゲームにログインした状態にします
2. **本ツールを起動する** — `STModuleManager.exe` をダブルクリック（自動で管理者権限が要求されます）
3. **ネットワーク監視をオンにする** — 画面下部ステータスバーの「ネットワーク監視」トグルをクリックしてオンにします
4. **スタレゾ内でエリア移動をする** — マップ移動やテレポートなどで場面を切り替えます

トグルをオンにすると「サーバー検出中...」と表示され、ゲームサーバーが見つかると「サーバー接続済み」に変わります。データ取得が不要なときはトグルをオフにしてください。

モジュールデータはエリア移動時の通信から取得しているため、新しいモジュールを入手した後にデータを更新したい場合は、本ツールを起動したまま再度エリア移動を行ってください。

### モジュール一覧・並び替え

ゲーム内では対応していないステータスごとの並び替えや、入手日時順・合計値順などのソートに対応しています。フィルタリングと組み合わせることで、目的のモジュールを素早く見つけることができます。

※ 入手日時はゲーム内のデータではなく、本ツールがモジュールを取得した日時が記録されます。

### 最適化

1. 画面上部のタブから「最適化」パネルを開く
2. **メインステータス**（必須）を選択 — +20到達を狙いたいステータス
3. **サブステータス**（希望）を選択 — 余裕があれば伸ばしたいステータス
4. 必要に応じて**除外ステータス**やレアリティフィルタを設定
5. 実行すると、スコア上位10件の組み合わせが表示されます

よく使う設定は「パターン保存」で名前を付けて保存・呼び出しができます。

## 最適化の計算方法

### 概要

手持ちモジュール（最大約2000個）から4つを選び、指定ステータスの**ブレイクポイント到達**と**+値合計**を最大化する組み合わせを求めます。

### ブレイクポイント

1つのステータスの合計値（4モジュール合算、最大+20）に応じて効果が段階的に上昇します。

| 合計値 | 効果 |
|--------|------|
| +1 | 発動 |
| +4 | 上昇 |
| +8 | 上昇 |
| +12 | 上昇 |
| +16 | 大幅上昇 |
| +20 | 最大 |

### スコア計算

各ステータスごとに、到達したブレイクポイントの**最高段階**に応じて点数が加算されます。

| ブレイクポイント | メイン（× 1.0） | サブ（× 0.3） | 非選択（× 0.15） | 除外 |
|-----------------|----------------|--------------|----------------|------|
| +20 到達 | 10,000 pt | 3,000 pt | 1,500 pt | 0 |
| +16 到達 | 5,000 pt | 1,500 pt | 750 pt | 0 |
| +12 到達 | 100 pt | 30 pt | 15 pt | 0 |
| +8 到達 | 50 pt | 15 pt | 7.5 pt | 0 |
| +4 到達 | 20 pt | 6 pt | 3 pt | 0 |
| +1 到達 | 5 pt | 1.5 pt | 0.75 pt | 0 |

これに加えて、**全ステータスの+値合計 × 2** がスコアに加算されます。

> **優先順位:** +20到達 > +16到達 > +値合計 > +12以下のブレイクポイント
>
> +20 / +16 の点数を+値ボーナスの理論最大値より十分大きく設定することで、ブレイクポイント到達が常に優先されます。

### 探索の高速化

C(2000, 4) ≈ 660億通りを全探索するのは現実的でないため、以下の手順でフィルタリングしています。

1. **関連性フィルタ** — メイン/サブステータスを1つも持たないモジュールを除外
2. **レアリティフィルタ** — 指定品質未満のモジュールを除外
3. **貢献度スコア Top N** — 各モジュールの貢献度（`Σ(メイン値×3) + Σ(サブ値×1) + Σ(その他値×0.5)`）上位300件に絞り込み

フィルタ後は150〜300個程度まで絞り込まれ、Rayonによる**マルチコア並列探索**と**枝刈り**（2つ選んだ時点で残りの各ステータスに+20を加算した楽観的上界スコアが現在のベストを下回る場合に打ち切り）を組み合わせることで、ギリギリCPU上での現実的な時間内での探索を可能にしています。
